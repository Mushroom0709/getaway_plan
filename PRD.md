# PRD — getaway_plan 旅行规划系统

## 目标

从 qinggan-travel（前端 SPA，所有数据预编译为 TS 文件）迁移到 getaway_plan（FastAPI 后端 + React 前端，数据存储在 MySQL 并通过 REST API 提供），实现可编辑、可持久化的旅行规划系统。

## IN_SCOPE

1. **旅行计划管理** — 创建/编辑/删除旅行项目（trips），含起止日期、状态
2. **行程日管理** — 每日行程 CRUD，按 day_number 排序
3. **地点标记** — 景点、住宿城市、沿途拍照点的地图标记及 CRUD，支持批量排序
4. **住宿推荐** — 酒店数据 CRUD（品牌、价格、房型、评分、图片）
5. **饮食推荐** — 餐厅 + 菜品 CRUD，每日餐食关联
6. **交通出行** — 航班、高铁、租车数据 CRUD
7. **路线规划** — 调用高德 REST API 计算驾车路线，存储 polyline
8. **景点详情** — 票价、营业时间、游玩时长、亮点、海拔、小贴士
9. **社交内容** — 小红书/抖音笔记及图片展示（静态预载数据）
10. **预算管理** — 费用类别/项目/单价/数量/小计 CRUD
11. **天气信息** — 各城市天气预报 CRUD
12. **地图展示** — 高德 JS API 2.0 地图，含标记、路线、InfoWindow
13. **认证** — 密码登录，JWT Bearer token 鉴权
14. **响应式** — 移动端 <768px 和桌面端 ≥768px 两套布局
15. **数据迁移** — 从 qinggan-travel 的 photos.ts、weather.ts、rental.ts 迁移到数据库
16. **批量照片导入** — scripts/photos.ts + photos_raw.json → 结构化入库

## OUT_OF_SCOPE

- 用户注册/多用户系统
- 实时导航/GPS 跟踪
- 社交分享/笔记发布
- 第三方 OAuth 登录
- 在线支付/预订功能
- 多语言国际化
- PWA/离线支持
- 高德 POI 实时搜索（数据由 qinggan-travel 预编译，手动维护）

## NICE_TO_HAVE

- AI 行程建议（调用外部 LLM）
- 基于用户评价的酒店/餐厅排序
- 导出 PDF/Excel 行程单
- 照片懒加载 + 虚拟滚动（当前 JS 包体积 ~1.4MB）
- 费用图表可视化

## 验收标准

1. 用户输入密码登录 → 获取 JWT token → 访问 API
2. 创建旅行 → 添加日程天 → 在每个天中添加景点 → 查看地图标记
3. 添加酒店/餐厅 → 添加餐厅的菜品 → 每日餐食关联
4. 调用高德路线规划 → 地图显示 polyline 方向箭头
5. 查看预算汇总、天气信息
6. 移动端 <768px 布局正常显示，地图 40vh
7. 所有 API 返回标准 JSON 响应
8. 数据可持久化到 MySQL
